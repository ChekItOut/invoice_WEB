**테스트 단계**:

4. ✅ PDF 파일 다운로드 확인
   - 파일명: `견적서_INV-002.pdf`
   - API 응답: 200 OK
   - 파일 형식: PDF document, version 1.3
   - 파일 크기: 12KB
   - 페이지 수: 1 page

**결과**: ✅ **통과**

### ✅ 시나리오 2: 404 에러 처리

**목표**: 존재하지 않는 견적서 ID 접근 시 에러 페이지 표시 검증

**테스트 단계**:
1. ✅ 잘못된 ID로 접근 (`http://localhost:3001/invoice/invalid-id-test-404`)
2. ✅ 에러 페이지 표시 확인
   - 제목: "오류가 발생했습니다"
   - 에러 메시지: "Notion API 오류: path failed validation: path.page_id should be a valid uuid, instead was `\"invalid-id-test-404\"`."
   - 안내 메시지: "문제가 지속되면 페이지를 새로고침하거나, 잠시 후 다시 시도해주세요."
3. ✅ 사용자 액션 버튼 확인
   - "다시 시도" 버튼

### ⚠️ 시나리오 4: API 에러 처리

**목표**: Notion API 에러 상황 대응 검증

**테스트 상태**: ⚠️ **수동 테스트 필요**

**이유**:
- 환경 변수 제거 후 서버 재시작이 필요하여 현재 진행 중인 개발 서버를 중단시킬 위험이 있음
- 실제 프로덕션 배포 전 수동으로 테스트 권장

**수동 테스트 절차**:
1. `.env.local` 파일에서 `NOTION_API_KEY` 제거 또는 잘못된 값으로 변경
2. 개발 서버 재시작 (`npm run dev`)
3. 견적서 페이지 접속
4. 에러 메시지 확인:
   - "서버 설정 오류: Notion API 키가 설정되지 않았습니다." 또는
   - "Notion API 인증 오류" 메시지 표시 확인
5. 환경 변수 복구 후 서버 재시작
